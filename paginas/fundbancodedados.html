SGBDs

Sistemas de gerenciamento de banco de dados definem como o banco vai funcionar.

Relacionais

Bancos relacionais parecem uma planilha do Excel. Têm colunas que definem os dados e linhas com entradas diferentes.

Não-relacionais

São mais flexíveis que os relacionais, guardam dados em chave:valor.

Exemplo: arquivo JSON


{
  "nome": "João",
  "idade": 30,
  "carro": null
}

Principais bancos

  • MongoDB

    • Banco não-relacional
    • Orientado a documentos
    • Suporta transações ACID

  • PostgreSQL

    • Banco relacional
    • Open source e sólido
    • Suporta JSON e extensões avançadas

  • Redis

    • Banco em memória
    • Estrutura chave-valor
    • Rápido para cache

Exemplo de diagrama MER

Exemplo de um sistema de biblioteca:

  • Entidades: Livro, Usuário, Empréstimo
  • Atributos:
    • Livro: código, título, autor, ano
    • Usuário: id, nome, endereço, telefone
    • Empréstimo: data_emprestimo, data_devolucao
  • Relações:
    • Usuário empresta Livro (1:n) - um usuário pega vários livros
    • Livro é emprestado por Usuário (n:1) - um livro vai para um usuário por vez
    • Empréstimo registra Livro e Usuário (1:1:1) - cada empréstimo liga um livro a um usuário

Dicionário de dados

É uma lista organizada que explica cada dado no sistema. Mostra nome da coluna, tipo de informação (texto, número, sim/não) e o que representa.

  • Texto - guarda palavras e frases como nomes e endereços
  • Número inteiro - números sem vírgula como idade e quantidade
  • Número decimal - valores com vírgula como preços e medidas
  • Booleano - só verdadeiro ou falso

Modelagem de banco de dados

MER

O Modelo entidade-relacionamento vem antes do DER. É só a ideia básica. Define entidades, atributos e relações, sem pensar em como implementar.

DER

O Diagrama entidade-relacionamento é o mapa do banco. Como na imagem abaixo, mostra visualmente como implementar o banco.

Entidades principais:

  • Academia - lugar que vende planos de assinatura
  • Plano_assinatura - pacotes com mensalidades e duração
  • Aluno - pessoa matriculada que compra planos e faz treinos
  • Treino - programa de exercícios feito por instrutores
  • Exercício - atividades que formam os treinos
  • Instrutor - profissional que cria treinos
  • Avaliação - notas e comentários sobre o plano do aluno

Relacionamentos:

  • Oferece (1:n) - uma academia vende vários planos
  • Contrata (1:n) - um aluno pode comprar vários planos com o tempo
  • Pertence (1:1) - cada aluno vai a uma academia
  • Participa (1:n) - um aluno faz vários treinos
  • Gera (1:1) - cada contrato cria uma avaliação
  • Contém (1:n) - um treino tem vários exercícios
  • Cria (1:n) - um instrutor faz vários treinos
