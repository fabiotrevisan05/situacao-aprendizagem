SGBDs

Os sistemas de gerenciamento de banco de dados são responsaveis por definiar a forma que o banco de dados bai trabalhar

Relacionais

Os bancos relacionais são muito parecidos com um Excel. Eles tem colunas e linhas onde as colunas definem qual dado será guardado e cada linha é uma entrada diferente na tabela.

Não-relacionais

São muito mais flexiveis que um banco relacional, permitindo armazenar dados com chave:valor

Ex: Arquivo JSON


{
  "nome": "João",
  "idade": 30,
  "carro": null
}

Principais Bancos

  • MongoDB

    • Banco não-relacional
    • Orientado a documentos
    • Suporta transações ACID

  • PostgreSQL

    • Banco relacional
    • Open source e robusto
    • Suporta JSON e extensões avançadas

  • Redis

    • Banco em memória
    • Estrutura chave-valor
    • Alta performance para cache

Dicionario de dados

É uma lista organizada que explica cada dado guardado no sistema. Mostra detalhes como nome da coluna, que tipo de informação aceita (texto, número, true/false) e para que serve cada campo.

  • Texto - armazena palavras e frases como nomes e endereços
  • Número inteiro - guarda números sem casas decimais como idade e quantidade
  • Número decimal - para valores com vírgula como preços e medidas
  • Booleano - armazena apenas verdadeiro ou falso

Modelagem de banco de dados

MER

O Modelo entidade-relacionamento é o que se faz antes de criar o DER. É apenas a idéia. Se define entidades, atributos, relaceos, mas sem pensar por enquanto na implementação.

DER

O Diagrama entidade-relacionamento é o "mapa" do banco de dados. Como na imagem abaixo, o DER vai ser a representação visual de implementação do banco de dados.

Entidades principais:

  • Academia - estabelecimento que oferece planos de assinatura
  • Plano_assinatura - pacotes com valores mensais e duração específica
  • Aluno - pessoa matriculada que contrata planos e participa de treinos
  • Treino - programa de exercícios criado por instrutores
  • Exercício - atividades específicas que compõem os treinos
  • Instrutor - profissional responsável por criar treinos
  • Avaliação - registro de notas e comentários sobre o plano do aluno

Relacionamentos:

  • Oferece (1:n) - uma academia oferece vários planos
  • Contrata (1:n) - um aluno pode contratar vários planos ao longo do tempo
  • Pertence (1:1) - cada aluno pertence a uma academia
  • Participa (1:n) - um aluno participa de vários treinos
  • Gera (1:1) - cada contratação gera uma avaliação
  • Contém (1:n) - um treino contém vários exercícios
  • Cria (1:n) - um instrutor cria vários treinos
